> Diese Datei spiegelt, was wir im Repo *heute* schon implementiert haben. Sie ist die Brücke zwischen "wie Claude Code funktioniert" (Files 00–03) und "was wir noch tun könnten" (File 05).

## 1. Die zwei Haupt-Konzepte unserer App

WhisperM8 ist primär eine **Diktier-App** (Push-to-Talk → Whisper-Transcription → Paste). Parallel haben wir eine **Agent-Chats-View** aufgebaut, in der wir Claude-Code-Sessions und Codex-Sessions je Projekt verwalten — dieselbe App-Datei, anderes Fenster.

Die Agent-Chats-View ist das, was der User "Agents View" nennt.

## 3. Datenmodell (kondensiert)

`AgentProvider`:
- `.codex` — startet `codex`
- `.claude` — startet `claude`

`AgentSessionKind`:
- `.chat` — normale interaktive Session
- `.agentView` — **`claude agents` Subcommand**, nur für Claude

`AgentChatSession`:
- `provider`, `projectID`, `externalSessionID?`, `title`, `model`, `reasoningEffort`
- `status` (pending/running/closed/archived), `color`, `groupName`, `sortIndex`
- `initialPrompt?`, `imagePaths[]`, `hasLaunchedInitialPrompt`, `shouldLaunchOnOpen?`
- `createdAt`, `lastActivityAt`, `lastTurnAt?`
- `createdManually?`, `titleIsAutoGenerated?`
- `summary?` (AgentSessionSummary mit headline + details + generatedAt + transcriptDigest)
- `kind?` (chat / agentView)

`AgentProject`:
- `id`, `name`, `path`, `color`, `lastBranch?`, `createdAt`, `updatedAt`
- `createdManually?`, `iconRelativePath?`, `customIconAbsolutePath?`, `iconAutoLookupAttempted?`, `sortIndex?`

`AgentWorkspace` = `{ schemaVersion, projects[], sessions[] }`. Persistiert atomisch.

## 4. Wie wir Claude / Codex starten

`AgentCommandBuilder.swift` → für jeden Tab:

**Codex (Chat):**
```bash
codex [extra-args] -C <project> -m <model> -c model_reasoning_effort=<effort> [--image <path>]* [<initialPrompt>]
# Resume:
codex resume [extra-args] -C <project> -m <model> -c model_reasoning_effort=<effort> <externalSessionID>
```

**Claude (Chat):**
```bash
claude [--settings <hook-bridge.json>] [extra-args] [--resume <uuid> | --session-id <uuid>] [<initialPrompt>]
```

**Claude (Agent View) — `.agentView`-Kind:**
```bash
claude agents [extra-args]
```
→ Keine `--session-id`, kein `--resume`, **keine** Hook-Bridge (denn das ist ein Dashboard, keine Single-Session).

Working-Dir = `project.path`. Keyboard-Profile passend zum Sub-CLI (`claudeCodeChat`, `codexChat`, `claudeAgentsView`).

## 5. PTY-Layer

`AgentTerminalView.swift` nutzt **SwiftTerm**'s `LocalProcessTerminalView`. Subklasse `QuietableTerminalView` deaktiviert Bell, falls Preference. `TerminalScrollGuard` fängt Mouse-Wheel-Events ab und übersetzt sie in **XTerm SGR Wheel-Bytes (Button 64/65)** ans PTY, wenn die TUI im Alt-Screen-Buffer ist — wichtig für die Agent-View-TUI von Claude (sonst könnte der User die Liste nicht scrollen + Sidebar wäre betroffen).

## 6. Hook-Bridge (`--settings`-Injection)

**`ClaudeHookSettingsBuilder.swift` + `ClaudeHookPaths` + `ClaudeHookBridge`:**

Pro lokaler WhisperM8-Session:
- Eine eigene Settings-JSON unter `~/Library/Application Support/WhisperM8/claude-hooks/<localID>.json`.
- Eine eigene Event-JSONL unter `~/Library/Application Support/WhisperM8/claude-session-events/<localID>.jsonl`.
- Settings registriert SessionStart + SessionEnd Hooks, die jeden Event-JSON-Blob in die Event-JSONL appenden.
- `ClaudeHookBridge` öffnet einen FD und registriert `DispatchSource.makeFileSystemObjectSource(.write|.extend)` → Kernel weckt uns, sobald Claude appended. **Idle-CPU = 0 %.**

Im aktuellen Setup wird das **nicht** für Agent View benutzt (`.agentView` skipt die Hook-Bridge, siehe `AgentSessionDetailView.swift` Z. 167).

## 7. Session-Discovery & Resume-Repair

- `AgentSessionIndexer.swift` scannt rekursiv `~/.claude/projects/**/*.jsonl` (+ analog Codex `~/.codex/sessions/`).
- Filtert `/subagents/` aus.
- Caches via `AgentSessionIndexCache` (per FileSize+mtime).
- Index liefert `IndexedAgentSession { externalSessionID, cwd, title, lastActivityAt, … }`.

`AgentSessionStore.repairResumeStateBeforeLaunch(...)`:
- **Fast Path** — Wenn die JSONL für die gespeicherte `externalSessionID` existiert: alles gut.
- **Slow Path** — Wenn nicht: Indexer scannt, `ClaudeActiveSessionResolver.decide(...)` matched anhand `cwd` + `launchedAt` + neueste `lastActivityAt` → `.unchanged | .rebound(newID) | .resetInvalid(oldID)` oder `.ambiguous(candidates)`.

→ Wir können also einen broken `--resume` heilen, falls Claude die ID intern umgebogen hat (`/resume` im TUI) oder der User das Transcript gelöscht hat.

## 8. Runtime-Watcher

`AgentSessionRuntimeWatcher` setzt einen `DispatchSource.makeFileSystemObjectSource` auf die JSONL einer laufenden Session. Daraus ableitet er einen feingranularen Live-Status (`AgentSessionRuntimeStatus`):

- `working` — JSONL kürzlich erweitert
- `awaitingInput` — Heuristik: Tool-Use offen + längere Ruhe
- `idle` — Process läuft, Antwort fertig, wartet auf neuen Prompt
- `stopped` — exit 0 oder kein Exit
- `errored` — exit ≠ 0

**Nicht persistiert** — existiert nur solange der Process lebt.

## 9. UI

- **Sidebar** mit Projekten (mit Icons, Group-Order, Branch-Anzeige) + Sessions je Projekt.
- **Top-Bar** mit Tab-Strip, "+" Menu (Neuer Codex Chat / Neuer Claude Chat / Neuer Claude Agent View).
- **Detail-View** rendert PTY (oder Empty-State, oder Transcript-Reader, wenn Session geschlossen).
- **Auto-Naming** via `AgentSessionAutoNamer` aus dem Transcript, sobald mindestens ein Turn fertig.
- **Auto-Summary** via `AgentSessionSummarizer` (im Code referenziert).
- **Drag-and-Drop** via `AgentDragDropPlanner`.
- **Resource-Monitor** zeigt CPU/RAM pro Session-Process.

## 11. Phase 6 (Roster-Import) — zurückgenommen

Eine frühere Build-Variante hat alle vom Supervisor gehosteten Background-Worker aus `~/.claude/daemon/roster.json` automatisch als `.backgroundChat`-Sessions in die Sidebar importiert (`SupervisorRosterReader` + `AgentSessionStore.importSupervisorWorkers`). Das hat **in der Praxis nicht gut funktioniert** — Sessions tauchten unter unklaren Projekten auf, die Sidebar wurde unübersichtlich, und der `claude agents`-TUI-Tab bleibt für die User-Übersicht ohnehin die offizielle Quelle.

**Rückbau:** `SupervisorRosterReader.swift`, der Import in `AgentSessionStore`, der Sonderfall im `sessions(for:)`-Filter und alle Roster-Tests sind entfernt. Eine Einmal-Migration `removeImportedBackgroundSessions(from:)` räumt die bereits in den Workspace gewanderten Roster-Imports (alle `.backgroundChat`-Sessions mit `createdManually != true`) beim nächsten Workspace-Load weg.
