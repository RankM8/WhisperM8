# Stores & persistence

Scope: `AgentSessionStore`, `TranscriptRunReportStore`, `AgentSessionIndexer`, `AppPreferences`, `ThemeManager`, `AgentChat`, `TranscriptContextBundle`, `PostProcessingTemplate`. Cross-referenced with `Tests/WhisperM8Tests/AgentChatsTests.swift`.

## Schema evolution & migration safety

`AgentWorkspace` (Models/AgentChat.swift:312-317) is a flat `{projects, sessions}` Codable struct with **no schema-version field**. Migration is implicit, structural, and forward-only.

`AgentProject` (Models/AgentChat.swift:89-148) now carries six optional "added later" fields: `createdManually`, `iconRelativePath`, `customIconAbsolutePath`, `iconAutoLookupAttempted`, `sortIndex`, `lastBranch`. All are correctly declared `Optional<T>` so a v1 workspace JSON missing them still decodes. Good. But because the decoder uses a synthesized `init(from:)`, **a non-optional addition would silently break the load** with a swallowed error (AgentSessionStore.swift:30-33 catches the decode failure and returns `.empty` — i.e. user loses their entire workspace silently). There is no integrity backup, no `.bak` rotation, nothing.

`AgentChatSession` (Models/AgentChat.swift:172-279) has 19 stored properties, 11 of them effectively optional. New-since-v1 fields that decode safely: `externalSessionID`, `color`, `groupName`, `sortIndex`, `initialPrompt`, `shouldLaunchOnOpen`, `createdManually`, `titleIsAutoGenerated`, `lastTurnAt`, `summary`. Two **non-optional** post-v1 fields are present and **will break old workspaces**:
- `imagePaths: [String]` (Models/AgentChat.swift:185)
- `hasLaunchedInitialPrompt: Bool` (Models/AgentChat.swift:186)

For any workspace JSON written before these fields were added, decoding the whole `AgentWorkspace` will fail and the user gets `.empty` (AgentSessionStore.swift:30-33). No test exercises this. The migration helper `migratedWorkspace` (AgentSessionStore.swift:533-538) only operates **post-decode** so cannot rescue this.

`AgentSessionSummary.transcriptDigest` (Models/AgentChat.swift:296) is optional → safe. `TranscriptContextBundle.agentChat` (Models/TranscriptContextBundle.swift:57) is optional → safe.

`IndexedAgentSession` (AgentSessionStore.swift:552-561) — `model`, `reasoningEffort` optional. Safe.

`TranscriptRunReportDraft` / `TranscriptRunReport` — only `Draft` exists in scope, but the `Report` it composes pulls in `replyIntent`, `visualManifest`, `agentProvider`, `agentSessionID`, `agentProjectPath` (TranscriptRunReportStore.swift:122-137). These all look optional in `Report` (writer side only), but bidirectional safety can't be verified from the scope here.

`AgentSessionIndexCache` is hand-coded `Codable` via `entries: [String: Entry]` (AgentSessionIndexer.swift:20). If the cache key format changes (`"\(provider.rawValue):\(path)"`, AgentSessionIndexer.swift:54-56), all entries silently miss — acceptable as cache, but no version handshake means a stale binary-format change would corrupt indefinitely until users delete the file.

## Concurrency model

`AgentSessionStore` is a `struct`, not an actor (AgentSessionStore.swift:3). Every mutator follows the pattern *read-whole-file → mutate → write-whole-file* (e.g. AgentSessionStore.swift:83-89, 147-153, 213-221).

**Risk: HIGH.** Two issues:

1. **Lost-update race.** If two callers fire `updateSession` concurrently (e.g. Runtime-Watcher writes `lastTurnAt` while the user renames the session), both read the same `workspace`, mutate disjoint fields, write back. The second writer overwrites the first writer's change, **including for unrelated sessions/projects** because the entire workspace is rewritten. The atomic-write (`options: .atomic`, AgentSessionStore.swift:50) protects the file from corruption but does **not** prevent lost updates — `.atomic` only guarantees readers never see a half-written file.

2. **No serialization primitive.** No `NSLock`, no actor, no serial `DispatchQueue` inside the store. Callers in the app may all hit MainActor (which would serialize), but `AgentSessionRuntimeWatcher` (referenced in code comments at AgentChat.swift:62) is a background process. The store is also passed to `AgentSessionSummarizer` (out-of-scope) — same risk applies.

The save cost is real: load+save are timed and logged (AgentSessionStore.swift:13, 39), so the team already knows latency matters at scale.

**No tests for concurrent writes.** The 1750-LOC test file covers single-threaded mutations only.

**Extraction opportunities:**
- The 6 "session metadata" mutators are all 2–5 lines wrapping `updateSession(id:_:)`. They exist for type-safety + clarity, but could be replaced by call-site closures with no real loss — most of them already delegate to `updateSession`. Keep them only if the call-site count justifies the symbol.
- Project icon trio (lines 109-134) could move into `AgentProjectIconStore` (the resolver service already exists — referenced in AgentChat.swift:99-113 comments). Split persistence by concern.
- Indexer integration (`bindLatestIndexedSession`, `mergeIndexedSessions`) is a different lifecycle (read-only-from-disk merge) — extract into `AgentSessionMerger` that takes a `AgentWorkspace` in/out, leaving the store as pure storage.
- Static helpers `sortedSessions`/`sortedProjects` could move to extensions on `[AgentChatSession]`/`[AgentProject]`.
- `currentGitBranch` (AgentSessionStore.swift:477-495) does a synchronous shell-out inside `upsertProject`. **Mis-belongs** — should be async / off the store entirely.

## AppPreferences

30 distinct preference keys in `PreferenceKeys` (AppPreferences.swift:222-253), each with a hand-written computed property of 3–8 lines. That's roughly 200 LOC of identical-shape getters and setters. Defaults are inlined inconsistently — e.g. `audioDuckingFactor` uses `value > 0 ? value : 0.2` (line 43-46), `maxScreenshotsPerRecording` clamps and falls back (lines 109-118), `boolWithDefault` exists as a helper (line 194-197) for some bools but not all.

**Migration logic:** exactly one (`migrateScreenshotLimitDefaultIfNeeded`, AppPreferences.swift:207-219). No versioning, no migration manifest. `overlayPositionX/Y` keys are declared (AppPreferences.swift:230-231) but **never accessed in this file** — dead key or read elsewhere.

**Defaults library proposal.** The SPM dep `sindresorhus/Defaults` is already in `Package.swift` per the CLAUDE.md doc. Migration would collapse most of this:

```swift
extension Defaults.Keys {
    static let language = Key<String>("language", default: "de")
    static let autoPaste = Key<Bool>("autoPasteEnabled", default: true)
    static let audioDuckingFactor = Key<Double>("audioDuckingFactor", default: 0.2)
    static let appearanceOverride = Key<AppearanceOverride>("appearanceOverride", default: .system)
    // ...
}
```

`Defaults` supports `RawRepresentable` keys natively, so the `appearanceOverride`, `defaultAgentProviderRaw`, `codexPostProcessingModelRaw` and friends become typed without hand-written get/set. The clamping logic for `maxScreenshotsPerRecording` would need a custom `Defaults.Serializable` wrapper or a thin facade — small price for losing ~150 LOC of boilerplate. The static `objectExists(for:)` / `removeObject(for:)` escape hatches (AppPreferences.swift:178-192) can stay during transition.

`AppPreferences.shared` is a `static var` (AppPreferences.swift:4) — surprising for a value-type singleton; tests can swap it which is fine, but using `var` invites accidental mutation. Make it `let` once tests can DI a `Defaults`-backed alternative.

## Indexer cache

`AgentSessionIndexCache.cacheKey` (AgentSessionIndexer.swift:54-56) is `"<provider.rawValue>:<standardizedPath>"`. **Invalidation predicate** (lines 44-52): cache hit only if both `fileSize` AND `modifiedAt` match. Both fields come from `FileMetadata` via URLResourceValues (lines 263-274, 431-442).

**Correctness concerns:**

1. The cache key omits `createdAt`. `FileMetadata.createdAt` is stored on the cache entry's input but NOT compared in `lookup` (AgentSessionIndexer.swift:46-48). If a Codex/Claude session file is replaced atomically (same path, same final size, same mtime — unlikely but possible on `cp -p`), the cache returns the old parse. Low likelihood; flagging for completeness.

2. The Claude indexer parses **up to 200 lines / 1 MiB** of the JSONL prefix (AgentSessionIndexer.swift:369). The `lastMessageDate` extracted may therefore stop short of the actual latest activity. But `metadata.modifiedAt` is also folded in via `max(...)` (line 417), so the cache key correctly invalidates when the file grows. Good.

3. Skipped/un-parseable files are explicitly cached as `nil` (AgentSessionIndexer.swift:217, 354) — prevents re-scan. Verified by `testCodexSessionIndexCacheRemembersSkippedFiles` (AgentChatsTests.swift:236). Good.

4. **No bound on cache growth.** Files removed from disk are never evicted from the on-disk cache; entries accumulate indefinitely. For users with thousands of historical sessions, this is a slow leak. Add a sweep step: drop entries whose path no longer exists on save.

5. Cache stored as JSON (AgentSessionIndexer.swift:74-103) — fine. No format version. Same risk as workspace store on schema change.

## TranscriptRunReportStore

`save` writes a directory-per-run: `Reports/<UUID>/{report.json, Attachments/...}` (TranscriptRunReportStore.swift:46-53). Attachments are **copied** (not moved or hardlinked, line 228) — original files survive. JSON write is atomic (line 144).

**Retention policy: NONE.** `recentReports(limit:)` (lines 148-164) only caps the *read* side at 50 by default. Nothing prunes old report directories. Every recording grows the reports folder unboundedly. Heavy users with `screenshot + clip` attachments will accumulate MB-per-recording → GB over time.

**Rotation: NONE.** No max-count, no max-age, no max-size enforcement. `delete(_:)` exists (lines 166-171) but is caller-driven.

**Sync vs async:** all I/O is synchronous in the calling thread. `save` does N attachment copies in a loop (line 56-62). For a recording with 20 screenshots (the default cap, AppPreferences.swift:7) this stalls the caller. No `Task.detached` or queue offload visible in scope.

**Risk severity: medium.** A storage-quota nuisance + UI stall after long recordings, not a data-loss bug.

## Data-type ergonomics

### `AgentChatSession`: 19 fields → grouping suggestion

Group icon/visual into a sub-struct:
```swift
struct SessionDisplay: Codable, Equatable, Hashable {
    var color: String?
    var groupName: String?
    var sortIndex: Int?
}
```
Group launch state:
```swift
struct SessionLaunchState: Codable, Equatable, Hashable {
    var initialPrompt: String?
    var imagePaths: [String]
    var hasLaunchedInitialPrompt: Bool
    var shouldLaunchOnOpen: Bool?
}
```
Group naming/auto-rename:
```swift
struct SessionTitleState: Codable, Equatable, Hashable {
    var title: String
    var titleIsAutoGenerated: Bool?
    var lastTurnAt: Date?
}
```

**Migration cost:** non-trivial — these groupings would break the on-disk JSON shape unless implemented behind custom `init(from:)` that reads both flat-and-nested. Probably defer until a versioned schema is in place. Worth noting up-front so the team plans for it.

### `AgentProject`: 12 fields → grouping suggestion

Three icon-related fields (`iconRelativePath`, `customIconAbsolutePath`, `iconAutoLookupAttempted`, lines 104-113) + `resolvedIconURL` accessor (line 155) clearly form a sub-domain:
```swift
struct ProjectIcon: Codable, Equatable, Hashable {
    var relativePath: String?
    var customAbsolutePath: String?
    var autoLookupAttempted: Bool?
    var resolvedURL: URL? { ... }
}
```
Same migration caveat applies.

### `PostProcessingTemplate.render` (Models/PostProcessingTemplate.swift:12-29) is 9 chained `replacingOccurrences` calls. Replace with a template engine or a dictionary loop. Not persistence per se but adjacent.

## Top findings ranked by severity

**hoch:**
1. `AgentChatSession.imagePaths` and `hasLaunchedInitialPrompt` are non-optional fields added after v1 (Models/AgentChat.swift:185-186). Old workspace JSONs fail to decode → user silently loses their entire workspace (AgentSessionStore.swift:30-33). Needs a default-injection custom decoder OR these fields become Optional, OR add a `.bak` rescue path.
2. No concurrency protection on `AgentSessionStore` mutators — full-workspace read-mutate-write race window between background watcher and UI mutations.
3. `TranscriptRunReportStore` has no retention policy → unbounded disk growth.

**mittel:**
4. No schema version field on `AgentWorkspace` or the indexer cache; any future incompatible field forces silent migration via failed decode.
5. Decode failure path swallows the error to `.empty` (AgentSessionStore.swift:30-33) — no `.bak` rotation, no user-visible warning.
6. `AppPreferences` boilerplate (~150 LOC) should migrate to the already-vendored `Defaults` library.
7. Indexer cache never evicts deleted files → slow disk leak.
8. `currentGitBranch` shell-out happens synchronously inside `upsertProject`.

**niedrig:**
9. `AgentChatSession` / `AgentProject` field count is unwieldy but works; grouping would help only if combined with a schema-version step.
10. Two dead `overlayPositionX/Y` keys (AppPreferences.swift:230-231).
11. Static helpers in `AgentSessionStore` could move to collection extensions.

## Tests needed before any refactor

1. **Decode a v1 workspace JSON that lacks `imagePaths` and `hasLaunchedInitialPrompt`.** Should decode without losing the rest of the workspace. (Not covered.)
2. **Decode a workspace JSON with an unknown extra field.** Should round-trip without losing the unknown field, or at minimum decode the known fields cleanly. (Not covered.)
3. **Decode a corrupted (truncated) workspace JSON.** Current behavior is silent `.empty` return — pin that or replace with `.bak` recovery. (Not covered.)
4. **Concurrent mutators on the same store.** Spawn two `Task`s racing `updateSession(idA)` + `updateSession(idB)`. Both updates must survive. (Not covered.)
5. **Concurrent writer + reader.** Reader must never see a partial workspace. `.atomic` should cover this — pin it. (Not covered.)
6. **`bindLatestIndexedSession` time-window edge cases** (`createdAt.addingTimeInterval(-5)`, AgentSessionStore.swift:367). The 5-second tolerance is undocumented and untested. (Not covered.)
7. **`TranscriptRunReportStore.save` with a vanished attachment source file.** `copyAttachment` returns `nil` (line 217) — does the report still save with `storedPath = nil`? Pin behavior. (Not covered.)
8. **Indexer cache eviction once a session file is deleted.** Currently no eviction — establish behavior before changing. (Not covered.)
9. **`AppPreferences.migrateScreenshotLimitDefaultIfNeeded` idempotence after the flag is set.** (Not covered.)
10. **`AgentSessionStore` against an unwritable directory.** Currently `loadWorkspace` returns `.empty`, `saveWorkspace` throws — pin error propagation. (Not covered.)
