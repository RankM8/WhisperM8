# Finding-Traceability-Matrix

Diese Matrix verknüpft jedes bestätigte Finding aus den Verdict-Quellen mit der Roadmap und dem dokumentierten Umsetzungsstand. Die vierfach wiederverwendeten Runde-3-IDs `G01` ff. werden mit den stabilen Roadmap-Aliasen `R3-DEF-*`, `R3-PROXY-*`, `R3-SEC-*` und `R3-MIX-*` dedupliziert. Die noch nicht verifizierte Terminal-Population erhält hier nur zur eindeutigen Rückverfolgbarkeit den Alias `R3-TERM-*`; sie ist **keine bestätigte G-Population**. `LÜCKE` bedeutet: keine belastbare Roadmap-Zuordnung.

Die bestätigte kritische/hohe Runde-4-Population ist seit dem 2026-07-20 vollständig in eigene kanonische Zeilen geroutet. Der Delta-Prüfstand vom 2026-07-19 gilt **nicht** für diese neuen Zeilen; für sie ist ausschließlich die eigene Spalte „Aktueller Status“ samt dortigem Commit-Nachweis maßgeblich.

Geprüfter Delta-Bereich: `e8b7661..HEAD` (19 Commits). Für einen Status „gefixt“ oder „teilgefixt“ wird ein Commit genannt; „offen“ ist zusätzlich an aktuellem Code belegt. Reine Zeilenverschiebungen durch neue Features gelten nicht als Fix.
