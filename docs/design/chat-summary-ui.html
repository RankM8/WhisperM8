1Subagent-Hierarchie

Auftrag → Ergebnis → Live-Verlauf. Der Auftrag (letzter Prompt an den Agent) steht fix oben als Bubble von links. Die Ergebnis-Karte hält ihren Platz in jedem Zustand — im Running-Zustand wohnt dort die Live-Zeile, das Ergebnis verdrängt nie den Verlauf.

2Report gerendert, nie JSON

Der Agent-Report wird als Headline + priorisierte Finding-Zeilen gerendert; der finale Turn-Text im Verlauf ebenso (der Screenshot-JSON-Blob ist das Anti-Muster). Chip Quelle: Agent-Report macht klar: hier lief kein LLM-Summary.

3Verlaufsladen = echter Button

Runde Pill-Form mit Rahmen, vier Zustände: Vorhanden (mit Fenster-Angabe „512 KB → 2 MB"), Lädt (Spinner, Button gesperrt), Nachgeladen („✓ 23 ältere Runden" + weiter-laden), Anfang erreicht (Trennlinien-Marker). Muss in Chat- UND Roh-Ansicht identisch funktionieren — der aktuelle Chat-Ansicht-Bug ist damit als Soll-Verhalten definiert.

4Markdown-Ruhe

Nummerierte Abschnitte mit dezenter Zahl, Bullets als 4-px-Punkte, Tabellen mit Hairlines + Uppercase-Kopf statt Monospace-Härte, code-Tokens leicht abgesetzt, H4-Abschnitte mit Trennlinie. Lange Berichte lesen sich wie ein Dokument, nicht wie Rohtext.

5Teammate-Block

Injizierte Teammate-/System-Nachrichten sind ein eigener gestrichelter Block: Badge + Ein-Zeilen-Gist immer sichtbar, das rohe Payload erst nach Klick. Nie wieder eine bildschirmfüllende JSON-Bubble.

6Letzter Prompt sichtbar

In Chats trägt jede Runde ihre Prompt-Bubble (auch die letzte); beim Subagent übernimmt der fixe Auftrag-Block diese Rolle. Zusammen mit „Runde unvollständig"-Markern geht der Bezug „worauf antwortet das?" nie verloren.
